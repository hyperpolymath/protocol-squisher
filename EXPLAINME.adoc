== Key Claims From README

[quote, README section "The Invariant"]
____
**The Invariant:** If it compiles, it carries.

Every data serialization format can talk to every other format. Always. Even if slow. Even if lossy. But it *will* transport.
____

*Evidence:* 13 protocol analyzers implemented and tested (Phase 2 complete, README line 20-30):
Rust (serde), Python (Pydantic), JSON Schema, Protobuf, Bebop, FlatBuffers, MessagePack, Avro, Cap'n Proto, Thrift, ReScript, GraphQL, TOML.

The transport classes guarantee (lines 102-124) specify:
- Concorde: Zero-copy, full fidelity (serde ↔ serde)
- Business: Minor overhead, full fidelity (Protobuf ↔ Thrift)
- Economy: Moderate overhead, documented losses (JSON ↔ MessagePack)
- Wheelbarrow: High overhead, significant losses, but guaranteed to work (Factor ↔ Cap'n Proto)

This is proven via property-based testing (937 tests passing, line 22) with coverage across all 13 analyzers plus Ephapax IR integration, optimizer, and Phase 3/4 modules.

*Caveat:* The "will transport" guarantee relies on JSON as the ultimate fallback (line 140) — if two formats cannot convert to each other, both convert to JSON. This is always possible but may be lossy (circular references, type precision loss).

[quote, README section "Phase 2 (Format Expansion - Complete)"]
____
**Diversity Spectrum Meta-Analysis** complete. **Squishability rankings** validated across design philosophies (zero-copy → schema evolution → dynamic).
____

*Evidence:* DIVERSITY-ANALYSIS.md (link line 194) provides meta-analysis of 11 protocols with rankings. Hypothesis testing confirms schema evolution protocols (Avro/Thrift) are most squishable, zero-copy (Cap'n Proto) already optimal (lines 29-30). This is documented in a standalone analysis file backed by empirical synthesis explorer database (line 159-162).

*Caveat:* The squishability analysis is relative to the 11 protocols studied. New protocols may have different rankings.
